\# Security Onion IDS Lab — Nmap Reconnaissance Detection



\## Project Overview



This project demonstrates the deployment of a Security Onion Intrusion Detection System (IDS) in a virtualized lab environment to detect network reconnaissance activity.



A simulated attacker system (Kali Linux) performed an Nmap scan against a vulnerable target system (Metasploitable). The Security Onion IDS successfully detected the reconnaissance activity and generated alerts, which were analyzed using Kibana dashboards.



This lab simulates a real-world SOC detection workflow including attack simulation, alert detection, and incident investigation.



\---



\## Objectives



\- Deploy a Security Onion IDS in VMware

\- Configure network monitoring interfaces

\- Simulate reconnaissance activity using Nmap

\- Detect malicious network activity using Suricata

\- Investigate alerts using Kibana dashboards

\- Document findings as part of a SOC analyst workflow

\## Attack Simulation



The attacker system executed an Nmap scan against the target machine to enumerate open ports and services.



Command executed:



nmap -A 192.168.247.138



This command performs:



\- OS detection

\- Service enumeration

\- Script scanning

\- Port scanning



\---



\## Detection



The Security Onion IDS detected reconnaissance activity generated by the Nmap scan.

192.168.247.138  



Destination Port:

Protocol:  

TCP  



Alert Signature:

Detection Engine:



Suricata



\---



\## Investigation



The alert was reviewed in the Security Onion dashboard and analyzed using Kibana.



The investigation confirmed:



\- The attacker system initiated multiple TCP connections

\- The target system received service enumeration requests

\- The IDS generated alerts based on known reconnaissance signatures

\- Network traffic matched Nmap scanning behavior
